# Storage Layer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy RustFS S3-compatible object storage on Kubernetes via Helm as the platform's raw data landing zone.

**Architecture:** Single `helm/storage/values.yaml` configures RustFS in standalone mode (one pod, one PVC) using the official `rustfs/rustfs` Helm chart. Credentials stored in a pre-created K8s secret following the project's existing pattern. `helm/README.md` updated with repo, secret, deploy, and access instructions.

**Tech Stack:** RustFS 1.0.0-beta.7, Helm chart 0.7.0 (`charts.rustfs.com`), Kubernetes, kubectl

---

### Task 1: Create `helm/storage/values.yaml`

**Files:**
- Create: `helm/storage/values.yaml`

- [ ] **Step 1: Create the values file**

```yaml
# RustFS standalone S3-compatible object storage
# Chart: rustfs/rustfs v0.7.0 (app: 1.0.0-beta.7)
# Docs: https://charts.rustfs.com

mode:
  standalone:
    enabled: true
  distributed:
    enabled: false

secret:
  existingSecret: "storage-rustfs-secret"  # keys: root-user, root-password

dataPersistence:
  size: 20Gi

buckets:
  - name: os-data-platform-ingestor

ingress:
  enabled: false

consoleIngress:
  enabled: false

resources:
  requests:
    cpu: 100m
    memory: 128Mi
  limits:
    cpu: 500m
    memory: 512Mi
```

Save to `helm/storage/values.yaml`.

- [ ] **Step 2: Validate YAML is well-formed**

```bash
helm repo add rustfs https://charts.rustfs.com
helm repo update
helm lint helm/storage/ --values helm/storage/values.yaml 2>/dev/null || \
  helm template storage rustfs/rustfs -f helm/storage/values.yaml --dry-run 2>&1 | head -20
```

Expected: no errors, or template output showing Deployment and PVC resources. If `helm lint` says chart directory not found, use `helm template storage rustfs/rustfs -f helm/storage/values.yaml --dry-run` directly.

- [ ] **Step 3: Commit**

```bash
git add helm/storage/values.yaml
git commit -m "feat: add RustFS storage helm values (standalone, 20Gi)"
```

## Post-Deploy Verification (manual)

After running `helm install storage rustfs/rustfs -f helm/storage/values.yaml -n os-data-platform`:

```bash
# Check pod is running
kubectl get pods -n os-data-platform -l app.kubernetes.io/name=rustfs

# Check PVC is bound
kubectl get pvc -n os-data-platform

# Access console
kubectl port-forward svc/storage-rustfs-console 9001:9001 -n os-data-platform
# Open http://localhost:9001 — log in with credentials from storage-rustfs-secret
# Verify bucket `os-data-platform-ingestor` exists
```

Airbyte S3 connector config:
- **Endpoint:** `http://storage-rustfs:9000`
- **Access Key:** value of `root-user` from `storage-rustfs-secret`
- **Secret Key:** value of `root-password` from `storage-rustfs-secret`
- **Bucket:** `os-data-platform-ingestor`
